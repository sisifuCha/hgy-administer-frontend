## GET 静态数据展示

GET /admin/getStaticData

获取本周的预约挂号人数，本周的就诊人数，不同科室的就诊人数，不同医生就诊人数，不同医生排班量

### 请求参数

|名称|位置|类型|必选|说明|
|---|---|---|---|---|
|Authorization|header|string| 否 |none|

> 返回示例

> 200 Response

```json
{
  "code": 0,
  "msg": "string",
  "data": {
    "regis_count": 0,
    "visitor_count": 0,
    "dep_visitor_count": [
      {
        "name": "string",
        "num": 0
      }
    ],
    "doc_visitor_count": [
      {
        "name": "string",
        "num": 0
      }
    ],
    "doc_schedule_count": [
      {
        "name": "string",
        "num": "string"
      }
    ]
  }
}
```

### 返回结果

|状态码|状态码含义|说明|数据模型|
|---|---|---|---|
|200|[OK](https://tools.ietf.org/html/rfc7231#section-6.3.1)|none|Inline|

### 返回数据结构

状态码 **200**

|名称|类型|必选|约束|中文名|说明|
|---|---|---|---|---|---|
|» code|integer|true|none||响应状态码|
|» msg|string|true|none||响应状态说明|
|» data|object|false|none||相应数据|
|»» regis_count|integer|true|none||本周预约挂号人数|
|»» visitor_count|integer|true|none||本周就诊人数|
|»» dep_visitor_count|[object]|true|none||本周各科室已就诊键值对数组|
|»»» name|string|true|none||科室名字|
|»»» num|integer|true|none||本周该科室已就诊人数|
|»» doc_visitor_count|[object]|true|none||本周医生看诊情况键值对|
|»»» name|string|true|none||医生名字|
|»»» num|integer|true|none||本周医生看诊人数|
|»» doc_schedule_count|[object]|true|none||本周医生排班情况键值对|
|»»» name|string|true|none||医生名|
|»»» num|string|true|none||本周医生排班数|

## GET 实时数据展示

GET /admin/getHotData

获取今日挂号人数，已经就诊的人数，正在排队的人数

> 返回示例

> 200 Response

```json
{
  "code": 0,
  "msg": "string",
  "data": {
    "register_num": 0,
    "visited_num": 0,
    "lining_num": 0
  }
}
```

### 返回结果

|状态码|状态码含义|说明|数据模型|
|---|---|---|---|
|200|[OK](https://tools.ietf.org/html/rfc7231#section-6.3.1)|none|Inline|

### 返回数据结构

状态码 **200**

|名称|类型|必选|约束|中文名|说明|
|---|---|---|---|---|---|
|» code|integer|true|none||none|
|» msg|string|true|none||none|
|» data|object|false|none||none|
|»» register_num|integer|true|none||今日挂号人数，即挂号表里所有今日的元组数量|
|»» visited_num|integer|true|none||今日已经就诊人数，即挂号表里今日status为已就诊的元组数|
|»» lining_num|integer|true|none||今日正在排队的患者人数，即挂号表里今日status为已挂号的元组数|

## GET 报表导出

GET /admin/getCSV

医生信息表，患者信息表，患者挂号记录表（筛选器是患者id和时间），医生排班表（筛选器是医生id和时间）

### 请求参数

|名称|位置|类型|必选|说明|
|---|---|---|---|---|
|type|query|string| 否 |DOC,PAT,REG,SCH分别对应医生信息表，患者信息表，挂号记录表和医生排班表|
|id|query|string| 否 |筛选器-人，这个参数是非必须的，即type为REG,SCH时候才启用|
|begin|query|string| 否 |筛选器-起始时间，这个参数是非必须的，即type为REG,SCH时候才启用，时间格式为YYYY-MM-DD，下同|
|end|query|string| 否 |筛选器-起始时间，这个参数是非必须的，即type为REG,SCH时候才启用|
|Authorization|header|string| 否 |none|

> 返回示例

> 200 Response

### 返回结果

|状态码|状态码含义|说明|数据模型|
|---|---|---|---|
|200|[OK](https://tools.ietf.org/html/rfc7231#section-6.3.1)|none|[res](#schemares)|
